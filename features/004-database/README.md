# T-004 — PostgreSQL 18.3 + Atlas Migrations

## Overview

Establish the database foundation: connection pooling, schema-as-code migrations with Atlas,
pgvector for embeddings, and the core schema conventions that every table in this project follows.

This spec defines the patterns. Every future migration inherits them.

---

## Stack

- **PostgreSQL 18.3** with `pgvector` extension
- **Atlas** (schema-as-code) — HCL schema files in `db/schema/`; no raw `CREATE TABLE` in migration files
- **pgx/v5** driver with `pgxpool` for connection pooling
- **sqlc** for type-safe query generation — all SQL in `db/queries/*.sql`

---

## Core Schema Conventions

These rules apply to **every user-owned table** in the project. Violations are bugs.

### Required Columns

```sql
-- Every user-owned table must have these columns
org_id      UUID        NOT NULL REFERENCES orgs(id) ON DELETE CASCADE
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
deleted_at  TIMESTAMPTZ                              -- soft delete; NULL = active
```

### Visibility (mandatory on all content nodes)

```sql
-- 4-value enum — defaults to 'personal', never escalates automatically
visibility  TEXT NOT NULL DEFAULT 'personal'
    CHECK (visibility IN ('personal', 'team', 'org', 'public'))
```

| Value | Who can read | How to set |
|---|---|---|
| `personal` | Only the owning user | Default for all new nodes |
| `team` | All members of the user's team in the org | Explicit "share to team" action |
| `org` | All org members — but de-attributed in org-layer queries | Explicit "contribute to org brain" |
| `public` | Anyone (future: public brain gardens) | Explicit publish; disabled by default |

**`personal` is always the default. The system never auto-escalates visibility.**
Publishing to `team` or `org` is an irreversible action (content is de-attributed at org layer).

### PHI Flag (mandatory on content nodes)

```sql
-- Set by AI classifier or user; triggers field encryption + audit logging
is_phi  BOOLEAN NOT NULL DEFAULT false
```

Content where `is_phi = true` is encrypted at rest (AES-256-GCM via T-104) and never returned
in list endpoints.

### Indexes Required on Every Table

```sql
-- Every foreign key column
CREATE INDEX ON <table> (org_id);

-- Every column used in a WHERE clause
-- Every column used in an ORDER BY
-- Composite index for the most common query pattern: (org_id, <filter_column>)
```

## Migration Strategy (Atlas)

Atlas uses HCL schema files as the source of truth. The workflow:

```bash
# Inspect current DB state and generate a migration plan
atlas schema diff --from "postgres://..." --to "file://db/schema" --format atlas

# Apply migrations
atlas migrate apply --url "postgres://..."

# Verify schema is in sync
atlas schema inspect --url "postgres://..." > current.hcl
diff current.hcl db/schema/
```

Migration files live in `db/migrations/` — generated by Atlas, not hand-written.
Every migration has an Up and a Down.

### Schema Files Layout

```
db/
├── schema/
│   ├── 001_core.hcl       ← orgs, users (set up in T-007/T-008)
│   ├── 002_nodes.hcl      ← nodes + edges (universal content graph)
│   ├── 003_events.hcl     ← domain_events (T-120)
│   ├── 004_audit.hcl      ← audit_log (T-104)
│   └── ...
├── queries/
│   ├── nodes.sql
│   ├── users.sql
│   └── ...
└── migrations/            ← Atlas-generated, never hand-edited
```

## Core Schema: Migration 001

### `org_units` table

The organizational hierarchy is stored as a self-referencing tree.
This table is the backbone of the recursive brain model.

```sql
CREATE TABLE org_units (
    id            UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id        UUID        NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
    parent_unit_id UUID       REFERENCES org_units(id) ON DELETE CASCADE,  -- NULL = root
    name          TEXT        NOT NULL,
    -- Free-form label — no enum constraint. Users name their own levels.
    -- Common examples: "team", "squad", "pod", "tribe", "circle", "chapter", "cell"
    -- The hierarchy is defined entirely by parent_unit_id, not by type names.
    unit_type     TEXT        NOT NULL DEFAULT 'unit',
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at    TIMESTAMPTZ
);

CREATE INDEX ON org_units (org_id);
CREATE INDEX ON org_units (parent_unit_id);
CREATE UNIQUE INDEX ON org_units (org_id, name, parent_unit_id);
```

Every org automatically has a root `org_unit` of type `org` (created on org signup).
Individual users are also wrapped in an implicit `org_unit` of type `individual`.
This means `unit_id` on nodes is always populated — there is no null case.

### `orgs` table

```sql
CREATE TABLE orgs (
    id             UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    name           TEXT        NOT NULL,
    slug           TEXT        NOT NULL UNIQUE,
    plan           TEXT        NOT NULL DEFAULT 'personal'
                               CHECK (plan IN ('personal', 'team', 'org', 'enterprise')),
    phi_enabled    BOOLEAN     NOT NULL DEFAULT false,  -- BAA signed; PHI mode active
    baa_signed_at  TIMESTAMPTZ,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at     TIMESTAMPTZ
);
```

### `users` table (abbreviated — full spec in T-008)

```sql
CREATE TABLE users (
    id             UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id         UUID        NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
    email          TEXT        NOT NULL UNIQUE,
    display_name   TEXT        NOT NULL,
    role           TEXT        NOT NULL DEFAULT 'member',
    password_hash  TEXT,                 -- argon2id hash; NULL for OIDC-only users
    pepper_version SMALLINT    NOT NULL DEFAULT 1,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at     TIMESTAMPTZ,
    last_active_at TIMESTAMPTZ
);

CREATE INDEX ON users (org_id);
CREATE INDEX ON users (email);
```

## Core Schema: Migration 002 — Nodes + Edges

This is the universal content model. All entities in Infinite Brain are nodes.

```sql
CREATE TABLE nodes (
    id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id          UUID        NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
    user_id         UUID        NOT NULL REFERENCES users(id) ON DELETE CASCADE,

    -- Content
    type            TEXT        NOT NULL,
    title           TEXT        NOT NULL,
    content         TEXT,
    content_enc     BYTEA,                -- AES-256-GCM ciphertext when is_phi = true

    -- Classification
    para            TEXT        CHECK (para IN ('project', 'area', 'resource', 'archive')),
    project_id      UUID        REFERENCES nodes(id) ON DELETE SET NULL,
    tags            TEXT[]      NOT NULL DEFAULT '{}',

    -- Organizational unit scope
    unit_id         UUID        NOT NULL REFERENCES org_units(id) ON DELETE CASCADE,

    -- Privacy + visibility (CORE — never omit, never change default)
    -- 'individual': only the owning user
    -- 'unit': all members of the owning unit_id
    -- 'unit_and_above': unit + all parent units up the hierarchy
    -- 'org': all org members (de-attributed at org layer)
    -- 'public': anyone
    visibility      TEXT        NOT NULL DEFAULT 'individual'
                                CHECK (visibility IN ('individual', 'unit', 'unit_and_above', 'org', 'public')),
    is_phi          BOOLEAN     NOT NULL DEFAULT false,

    -- Embeddings + search
    embedding       VECTOR(1536),
    search_vector   TSVECTOR    GENERATED ALWAYS AS (
                        to_tsvector('english', coalesce(title, '') || ' ' || coalesce(content, ''))
                    ) STORED,

    -- Spaced repetition (T-036)
    review_stage    SMALLINT    NOT NULL DEFAULT 0,
    next_review_at  TIMESTAMPTZ,

    -- Deduplication (T-128)
    dedup_dismissed BOOLEAN     NOT NULL DEFAULT false,

    -- Standard columns
    metadata        JSONB       NOT NULL DEFAULT '{}',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at      TIMESTAMPTZ,
    archived_at     TIMESTAMPTZ
);

-- Indexes
CREATE INDEX ON nodes (org_id, user_id);
CREATE INDEX ON nodes (org_id, type);
CREATE INDEX ON nodes (unit_id);
CREATE INDEX ON nodes (org_id, visibility);          -- visibility-scoped queries
CREATE INDEX ON nodes (org_id, user_id, para);
CREATE INDEX ON nodes (project_id);
CREATE INDEX ON nodes (org_id, user_id, review_stage, next_review_at);
CREATE INDEX ON nodes USING hnsw (embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 64);             -- HNSW for fast ANN search
CREATE INDEX ON nodes USING gin (search_vector);     -- full-text search
CREATE INDEX ON nodes USING gin (tags);              -- tag filtering

-- Constraint: PHI content must be encrypted (enforced in application layer, documented here)
-- If is_phi = true, content must be NULL and content_enc must be set.
```

```sql
CREATE TABLE edges (
    id            UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id        UUID        NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
    from_node_id  UUID        NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
    to_node_id    UUID        NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
    relation_type TEXT        NOT NULL,
    confidence    FLOAT       NOT NULL DEFAULT 1.0 CHECK (confidence BETWEEN 0.0 AND 1.0),
    created_by    TEXT        NOT NULL CHECK (created_by IN ('user', 'ai', 'insight-linker')),
    metadata      JSONB       NOT NULL DEFAULT '{}',
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (from_node_id, to_node_id, relation_type)
);

CREATE INDEX ON edges (org_id);
CREATE INDEX ON edges (from_node_id);
CREATE INDEX ON edges (to_node_id);
CREATE INDEX ON edges (org_id, relation_type);
```

### Edge visibility rule

Edges do not carry a visibility field. An edge is visible if and only if the requesting user
has read access to **both** the `from_node` and `to_node`. This is enforced in the service
layer, not the query layer.

## Row-Level Security

RLS enforces org isolation at the database level — a compromised application cannot leak
another org's data even with unrestricted SQL.

```sql
-- Enable RLS on all user-owned tables
ALTER TABLE nodes ENABLE ROW LEVEL SECURITY;
ALTER TABLE edges ENABLE ROW LEVEL SECURITY;

-- Application role can only see rows belonging to its current org
-- org_id is set per-connection via SET LOCAL app.current_org_id = '<uuid>'
CREATE POLICY nodes_org_isolation ON nodes
    USING (org_id = current_setting('app.current_org_id')::uuid);

CREATE POLICY edges_org_isolation ON edges
    USING (org_id = current_setting('app.current_org_id')::uuid);
```

The application sets `app.current_org_id` at the start of every request via the auth middleware.

## Visibility Query Patterns

The service layer enforces visibility on every query. Never query nodes without a visibility filter.

```go
// internal/nodes/repository_pg.go

// Personal query: only the user's own nodes
func (r *NodeRepo) ListPersonal(ctx context.Context, userID uuid.UUID) ([]*Node, error) {
    // WHERE org_id = $1 AND user_id = $2 AND visibility = 'personal' AND deleted_at IS NULL
}

// Team query: personal + team-shared nodes within the org
func (r *NodeRepo) ListForTeam(ctx context.Context, orgID uuid.UUID, userID uuid.UUID) ([]*Node, error) {
    // WHERE org_id = $1 AND (user_id = $2 OR visibility IN ('team', 'org'))
    // AND deleted_at IS NULL
}

// Org query (anonymized): used ONLY by the org intelligence layer
// Returns anonymized aggregates, never raw nodes with user attribution
func (r *NodeRepo) AggregateForOrg(ctx context.Context, orgID uuid.UUID, q AggQuery) ([]OrgAggregate, error) {
    // COUNT, cluster, aggregate — user_id never returned
    // Filtered by q.MinGroupSize (k-anonymity)
}
```

## Connection Pooling

```go
// pkg/database/postgres.go

func New(cfg Config) (*pgxpool.Pool, error) {
    config, err := pgxpool.ParseConfig(cfg.DSN)
    config.MaxConns        = int32(cfg.MaxConns)       // default: 25
    config.MinConns        = int32(cfg.MinConns)       // default: 5
    config.MaxConnLifetime = cfg.MaxConnLifetime        // default: 1h
    config.MaxConnIdleTime = cfg.MaxConnIdleTime        // default: 30m

    // Register pgvector type for vector columns
    config.AfterConnect = func(ctx context.Context, conn *pgx.Conn) error {
        return pgvector.RegisterTypes(ctx, conn)
    }

    return pgxpool.NewWithConfig(ctx, config)
}
```

## Make Targets

```makefile
db-migrate:       ## Apply pending Atlas migrations
    atlas migrate apply --url "$(DATABASE_URL)"

    atlas schema diff --from "$(DATABASE_URL)" --to "file://db/schema"
```
