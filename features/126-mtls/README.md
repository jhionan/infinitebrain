# T-126 — mTLS Internal Service Communication

## Overview

Internal gRPC connections between services use mutual TLS. Every service presents a
certificate. Services without a valid certificate are rejected even inside the VPC.
Zero Trust networking: nothing is trusted by virtue of being on the internal network.

## Certificate Strategy

Dev: self-signed certificates generated by `make gen-certs` (stored in `certs/`, gitignored).
Production: certificates issued by an internal CA managed by OpenBao's PKI secrets engine.
OpenBao auto-rotates certificates before expiry.

## Implementation

```go
// pkg/grpc/mtls.go

func ServerTLSConfig(certFile, keyFile, caFile string) (*tls.Config, error) {
    cert, err := tls.LoadX509KeyPair(certFile, keyFile)
    if err != nil {
        return nil, fmt.Errorf("loading server cert: %w", err)
    }
    caCert, err := os.ReadFile(caFile)
    if err != nil {
        return nil, fmt.Errorf("loading CA cert: %w", err)
    }
    caPool := x509.NewCertPool()
    caPool.AppendCertsFromPEM(caCert)

    return &tls.Config{
        Certificates: []tls.Certificate{cert},
        ClientCAs:    caPool,
        ClientAuth:   tls.RequireAndVerifyClientCert,  // mutual TLS
        MinVersion:   tls.VersionTLS13,
    }, nil
}

func ClientTLSConfig(certFile, keyFile, caFile string) (*tls.Config, error) {
    // Same pattern — client also presents a cert
}
```

```go
// cmd/server/main.go

tlsCfg, err := grpcmtls.ServerTLSConfig(
    cfg.TLS.CertFile,
    cfg.TLS.KeyFile,
    cfg.TLS.CAFile,
)
grpcServer := grpc.NewServer(
    grpc.Creds(credentials.NewTLS(tlsCfg)),
    // ... interceptors
)
```

## OpenBao PKI Integration (Production)

```bash
# OpenBao issues short-lived certificates (24h TTL) per service
bao secrets enable pki
bao write pki/roles/infinite-brain-service \
    allowed_domains="svc.infinitebrain.internal" \
    allow_subdomains=true \
    max_ttl="24h"
```

Services fetch fresh certificates on startup and rotate before expiry via the
`SecretManager.Watch` interface from T-104.

## Makefile

```makefile
gen-certs:
    @mkdir -p certs
    # Generate CA
    openssl req -x509 -newkey rsa:4096 -keyout certs/ca.key -out certs/ca.crt \
        -days 365 -nodes -subj "/CN=infinite-brain-dev-ca"
    # Generate server cert signed by CA
    openssl req -newkey rsa:4096 -keyout certs/server.key -out certs/server.csr \
        -nodes -subj "/CN=server.svc.infinitebrain.internal"
    openssl x509 -req -in certs/server.csr -CA certs/ca.crt -CAkey certs/ca.key \
        -CAcreateserial -out certs/server.crt -days 365
```

## Acceptance Criteria

- [ ] gRPC server requires client certificate (mTLS)
- [ ] gRPC client presents certificate on every connection
- [ ] Connection without valid cert rejected with `codes.Unavailable`
- [ ] `make gen-certs` generates dev certs locally
- [ ] `certs/` in .gitignore
- [ ] Production path: OpenBao PKI issues and rotates certs automatically
- [ ] TLS 1.3 minimum enforced
